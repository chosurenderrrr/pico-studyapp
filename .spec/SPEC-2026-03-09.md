# SPEC - 技術仕様・要件定義

## 機能概要
以下の2機能を追加・変更する。
1. **タイトル機能**：問題ごとに任意のタイトルを設定でき、問題選択画面に表示する
2. **選択問題の刷新**：固定4択を廃止し、正解プール（複数可）＋誤答プールから構成する複数選択問題に変更

整序問題（`order` タイプ）は前サイクルで実装済み・変更なし。

---

## データ構造

### ファイル構成（問題フォルダ内）

```
[quiz-set-name]/
└── 001/
    ├── title.txt             ← タイトル（任意、全問題タイプ共通）
    ├── question.txt          ← 問題文（既存と同じ）
    ├── question_image.png    ← 問題画像（任意、既存と同じ）
    │
    │── [multiple-choice のみ]
    ├── correct1.txt          ← 正解プール（1〜N個）※出題時に全件表示
    ├── correct2.txt
    ├── ...
    ├── wrong1.txt            ← 誤答プール（1〜N個）
    ├── wrong2.txt
    ├── ...
    ├── display_count.txt     ← 出題時の選択肢表示数合計（例: "5"）
    │                           ※ display_count = 正解数 + 表示する誤答数
    │                           ※ 正解数 < display_count ≤ 正解数 + 誤答数
    │
    │── [order のみ（変更なし）]
    ├── type.txt              ← "order"
    ├── item1.txt 〜 item20.txt
    │
    └── explanation.txt       ← 解説（任意、既存と同じ）
```

**旧フォーマット（`correct.txt` / `wrong1〜3.txt`）は廃止**。既存データとの互換性は不要。

### メモリ上の問題オブジェクト

```js
{
  num, handle,
  title: '',                          // タイトル（空文字 = 未設定）
  problemType: 'multiple-choice' | 'order',

  // multiple-choice
  questionType, question,
  corrects: [{ type, content }],      // 正解プール（全件出題時に表示）
  wrongs:   [{ type, content }],      // 誤答プール
  displayCount: 4,                    // 出題時の表示数合計
  explanationType, explanation,

  // order（変更なし）
  questionType, question,
  items: [{ type, content }],
  explanationType, explanation,
}
```

## ProblemIO 変更

### load()

1. `title.txt` を読み込み `title` にセット（なければ空文字）
2. `type.txt` が `"order"` → `problemType = 'order'`、`item1〜20.txt` を読み込む（変更なし）
3. それ以外 → `problemType = 'multiple-choice'`
   - `correct1.txt`〜`correctN.txt` を連番で読み込み `corrects[]` に格納（存在しないところで終了）
   - `wrong1.txt`〜`wrongN.txt` を同様に読み込み `wrongs[]` に格納
   - `display_count.txt` を読み込み `displayCount` にセット
     （なければデフォルト = `corrects.length + wrongs.length` = 全表示）

### save()

1. `title.txt` を書き込む（空文字の場合はスキップ）
2. `order` の場合：変更なし
3. `multiple-choice` の場合：
   - `correct1.txt`〜`correctN.txt` を書き込む
   - `wrong1.txt`〜`wrongN.txt` を書き込む
   - `display_count.txt` を書き込む

## エディタ（Editor）変更

### タイトル入力欄
- エディタ最上部に「タイトル（任意）」テキスト入力を追加
- `title.txt` と連動

### 選択問題エディタ UI（旧4択エディタを置き換え）

```
[ 問題タイプ: 選択問題 | 整序問題 ]
[ タイトル（任意）: _____ ]

[ 問題文 テキスト/画像 ] ... 既存と同じ

[ 正解の選択肢（全件が出題時に表示される）]
  ① [テキスト入力]  [✕]
  ② [テキスト入力]  [✕]
  [ ＋ 正解を追加 ]

[ 誤答の選択肢 ]
  ① [テキスト入力]  [✕]
  ② [テキスト入力]  [✕]
  [ ＋ 誤答を追加 ]

[ 出題時の表示数: [数値入力] 個 ]
  ※ 正解数 < 表示数 ≤ 正解数 + 誤答数

[ 解説（任意） ] ... 既存と同じ
```

### バリデーション
- 正解が0個 → 保存不可（エラートースト）
- 誤答が0個 → 保存不可（エラートースト）
- 表示数 ≤ 正解数 → 保存不可「表示数は正解数より大きくしてください」
- 表示数 > 正解数 + 誤答数 → 保存不可「表示数がプールの合計を超えています」

## クイズ画面（Quiz）変更

### 選択問題の出題フロー

1. `corrects[]` を全件取得（正解の選択肢として全員表示）
2. `wrongs[]` から `displayCount - corrects.length` 個をランダム選択（誤答として表示）
3. 合計 `displayCount` 個の選択肢をシャッフルして表示
4. ユーザーは複数選択可能（タップでトグル）
5. 「回答確定」ボタンを押すと判定

### 複数選択 UI

```
[ 問題文 ]

[ 選択肢（シャッフル表示）]
  [A] → タップで選択（選択中はハイライト）
  [B] → タップで選択
  [C] → 未選択
  ...

[ 回答確定ボタン ] ← 1つ以上選択したら有効
```

### 正誤判定
- 選択した選択肢のセット = `corrects[]` の全件 と完全一致 → 正解
- 1つでも違えば不正解

### フィードバック表示
- 正解の場合：「正解！」表示 + 解説（あれば）
- 不正解の場合：各選択肢を色分け表示
  - 選択した正解：緑
  - 選択した誤答：赤
  - 未選択の正解：黄（選ぶべきだった）
  - 未選択の誤答：グレー（そのまま）

### 統計記録
- `Data.recordResult()` は既存と同様に呼び出す（完全一致で isCorrect = true）

## 問題選択画面（App）変更

- 問題リストの各アイテムにタイトルを表示
  - `title` が設定されている場合：タイトルのみ表示
  - `title` が未設定の場合：問題文の先頭20文字を表示（従来通り）
- 整序問題アイコン（🔢）は引き続き表示
