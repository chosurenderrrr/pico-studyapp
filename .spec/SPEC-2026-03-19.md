# SPEC - 技術仕様・要件定義

## 機能概要

### 画像リサイズ機能
- 問題登録時に画像を自動リサイズ
- 最大サイズ: 幅600px × 高さ無制限（縦長画像を許容）
- アスペクト比を維持（縦横比を保ったまま縮小）
- 画像形式: JPEG/PNG維持、base64形式で保存
- リサイズ処理はCanvas APIを使用（外部ライブラリ不使用）

### 問題作成UIの改善
- 問題作成画面開時に1ページ目をデフォルトで表示（追加ボタン不要）
- 回答項目表示領域をスクロール対応（1画面内に収める必要なし）

### GitHub連携機能
#### 概要
- 作成した問題セットをGitHubリポジトリにアップロード・ダウンロードできる
- 自分向けのWeb共有手段として利用（複数デバイスからアクセスするため）
- 既存のローカルFile System API保存は残し、GitHub連携は追加機能として共存

#### 認証
- **Personal Access Token（PAT）** をアプリ内で入力・保存
- PATは `localStorage` に保存（`gh_token` キー）
- 必要スコープ：`repo`（プライベートリポジトリも扱う場合）または `public_repo`

#### リポジトリ管理
- PAT入力後、GitHub API でリポジトリ一覧を取得しドロップダウン表示
  - エンドポイント：`GET /user/repos`（自分のリポジトリ一覧）
- リポジトリの新規作成も可能（名前・public/private を指定）
  - エンドポイント：`POST /user/repos`
- 選択したリポジトリは `localStorage` に保存（`gh_repo` キー：`owner/repo` 形式）

#### データ構造（リポジトリ内）
- 問題セットは `quiz-sets/` ディレクトリ内に保存
- ファイル名：`{問題セット名}.json`（例：`quiz-sets/英語基礎.json`）
- ファイル内容：既存の問題セットJSONデータをそのまま使用

#### アップロード（ローカル → GitHub）
- 問題セット詳細画面の ⋮ メニューからアップロード操作が可能
- エンドポイント：`PUT /repos/{owner}/{repo}/contents/{path}`
- 同名ファイルが存在する場合は更新（SHAを取得して上書き）
- 成功/失敗をトースト通知で表示

#### ダウンロード（GitHub → ローカル）
- GitHub上の問題セット一覧を取得して表示
  - エンドポイント：`GET /repos/{owner}/{repo}/contents/quiz-sets`
- 1ファイルずつダウンロードしてローカルに追加可能
  - エンドポイント：`GET /repos/{owner}/{repo}/contents/{path}`
- ダウンロード後は既存の問題セット一覧に追加

#### スコープ外（今回は対象外）
- 学習ログ・統計データ（study-log, quiz-stats）のGitHub保存
- 複数ユーザー間の共有・コラボレーション
- 自動同期（手動操作のみ）

- 外部ライブラリ禁止
- Android Chrome タッチ操作対応
