スマートフォン利用前提: UIはモバイルファースト設計
単一HTMLファイル: index.html 1ファイルに CSS・JS をすべてインライン実装
ローカルフォルダが主: File System Access API でデバイス上のフォルダを読み書き

[workspace]/               ← 初回起動時にユーザーが選択（File System Access API）
├── data/                  ← アプリが自動生成
│   ├── study-log.json
│   ├── quiz-stats.json
│   └── calendar.json
└── [quiz-set-name]/       ← クイズセット（複数可）
    └── 001/
        ├── question.png/txt
        ├── correct.png/txt
        └── wrong1.png/txt （wrong2, wrong3 も可）

問題を編集することができる
問題は作成、削除、移動が行える
問題の作成は画像、テキストで行う
問題の作成時の画像は画像を読み込んで簡単な編集ができるようにするトリムや拡大縮小　黒ペンツールで入力や追加で読み込んだ画像との重ね合わせなど

また、解答も問題と併せて作成できるものとする
解答の作成は問題の作成の仕様と同じようにする
解答には解説も含む

問題セットごとに苦手問題のみを出すモードも必要
問題だけを出す機能と前回間違えた問題だけを出す機能を含むこと

またカレンダーも実装すること　カレンダーに予定を入力できるようにする　また問題の正答率やその他統計データをいい感じにカレンダーとセットで見れる機能を表示非表示が切り替えられるようにしてじっそうすること
