# SPEC - 技術仕様・要件定義

## 機能概要
1. 問題読み込み・保存処理の高速化（並列化）
2. 選択問題の「出題時の表示数」設定を廃止し、全選択肢を常に表示
3. 整序問題にダミー選択肢（不正解）を追加できるようにする
4. 問題作成画面の保存ボタンをコンテンツ末尾にも配置

---

## 1. 処理の高速化

### 問題読み込み（ProblemIO.loadAll）
- 現状：問題ごとに順番（逐次）で複数ファイルを読み込んでいる
- 改善：`Promise.all` で全問題を並列読み込みに変更
- `ProblemIO.load` 内部の複数ファイル読み込みも並列化できる箇所は並列化

### 問題保存（ProblemIO.save）
- 現状：ファイルを1件ずつ順番に書き込んでいる
- 改善：独立した書き込み処理を `Promise.all` で並列化

---

## 2. 「出題時の表示数」廃止

### エディタ変更
- 「出題時の表示数」入力欄（`#mc-display-count-section`）を削除
- `display_count.txt` は保存・読み込みともに廃止

### クイズ動作変更
- 選択問題の選択肢 = 正解プール全件 ＋ 誤答プール全件（ランダム順）
- `displayCount` プロパティを廃止、代わりに全選択肢を常に使用

### 後方互換性
- 既存の `display_count.txt` が存在してもロード時に無視する（読み込まない）

---

## 3. 整序問題にダミー選択肢を追加

### データ構造変更
- 現状：`items[]`（正解順のリスト）のみ
- 変更後：
  - `items[]`：正解の選択肢（並び替える項目、上から正解順）
  - `dummies[]`：不正解のダミー選択肢（並び替えに使わないもの）

### ファイル構造
- `item1.txt` ～ `itemN.txt`：正解選択肢（既存のまま）
- `dummy1.txt` ～ `dummyN.txt`：ダミー選択肢（新規追加）

### エディタ UI
- 既存の「選択肢（上から順番が正解順）」セクションはそのまま維持
- 新たに「ダミーの選択肢（不正解）」セクションを追加
  - 「＋ ダミーを追加」ボタンで追加（上限20件）
  - ダミーは空でも問題なし（0件も可）

### クイズ動作
- スロット数 = `items.length`（正解選択肢の数）
- 出題時の選択肢 = `items[]` ＋ `dummies[]` をランダムシャッフル
- ユーザーはシャッフルされた選択肢からスロットに選んで当てはめる
- 採点：スロット1〜N が `items[0]`〜`items[N-1]` と一致で正解

---

## 4. 保存ボタンをコンテンツ末尾に追加

### エディタ変更
- 「この問題を削除」ボタンの上（または下）に「保存」ボタンを追加
- スタイルはヘッダーの保存ボタンと同じ動作（`Editor.save()` 呼び出し）
- 新規作成時・編集時いずれも表示

---

## 非機能要件（継続）
- 単一HTMLファイル（`index.html`）にインライン実装
- 外部ライブラリ禁止
- Android Chrome タッチ操作対応
