# SPEC - 技術仕様・要件定義

## 機能要件

### 1. ワークスペース管理
- 初回起動時に File System Access API でローカルフォルダを選択
- 選択フォルダをワークスペースとして使用（以降は自動記憶）
- `data/` フォルダをアプリが自動生成・管理

### 2. クイズセット管理
- フォルダ構造でクイズセットを管理
  ```
  [workspace]/
  ├── data/
  │   ├── study-log.json
  │   ├── quiz-stats.json
  │   └── calendar.json
  └── [quiz-set-name]/
      └── 001/
          ├── question.png または question.txt
          ├── correct.png または correct.txt
          ├── wrong1.png または wrong1.txt
          ├── wrong2.png または wrong2.txt（任意）
          └── wrong3.png または wrong3.txt（任意）
  ```
- 選択肢数：正解1つ＋不正解1〜3つ（最大4択）
- 問題の作成・編集・削除・移動が可能

### 3. 問題・解答の作成
- **コンテンツ形式**：テキスト または 画像（PNG/JPG）
- **MVP**：画像は読み込みのみ（表示・保存）
- **将来実装**：トリム・拡大縮小・黒ペン書き込み・画像重ね合わせ
- 解答には解説を含められる（テキスト or 画像）

### 4. クイズ出題モード
- **通常モード**：セット内の全問題をランダム出題
- **苦手モード**：正答率が低い問題のみ出題
- **前回ミスモード**：前回セッションで間違えた問題のみ出題

### 5. 統計・学習ログ
- 問題ごとの正解数・不正解数・正答率を記録
- セッションごとの学習ログを保存（日時・問題数・正答率）
- `quiz-stats.json` に蓄積

### 6. カレンダー
- 月表示カレンダー
- 日付に対してメモ・予定を入力可能（自由記述）
- 各日の学習統計（出題数・正答率）をカレンダー上に表示
- 統計表示は表示/非表示を切り替え可能
- `calendar.json` に保存

---

## 非機能要件

- **プラットフォーム**：Android Chrome をメインターゲット
- **UIデザイン**：モバイルファースト（タッチ操作前提、大きめのボタン）
- **オフライン動作**：ローカルのみで完結（サーバー不要）
- **パフォーマンス**：問題数1000件以上でも動作すること

---

## 技術構成

- **実装形式**：単一 HTML ファイル（`index.html`）に CSS・JS をすべてインライン
- **外部ライブラリ**：禁止（ゼロ依存）
- **ストレージ**：File System Access API（ローカルフォルダ直接読み書き）
- **データ形式**：JSON（study-log / quiz-stats / calendar）
- **画像対応**：PNG / JPG
- **ブラウザAPI**：File System Access API・Canvas 2D（将来の画像編集用）
